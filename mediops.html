Problem

Healthcare drowns in complexity

Complex workflows, cluttered interfaces, poor data handling. Hospital software built by engineers — not the nurses, doctors, and admins who use it every day.

Solution

Dashboard-first, role-aware

Each role — Admin, Doctor, Patient — sees exactly what they need on login. No deep nav trees. No training manuals. Just immediate, relevant clarity.

Three Roles, One System

Built for each user

Admin

Staff and appointment oversight

Doctor

Patient records and scheduling

Patient

Appointments and health data

Core Features

What it does

  • Role-based access — each view built independently
  • Appointment and patient data management
  • Scannable dashboards with strong hierarchy
  • Efficient workflows — minimum steps per action

"Admin needs depth. Doctors need speed. Patients need simplicity. Strict view isolation — each role's experience is architecturally separate, sharing data but never interface patterns."

Outcome

Clearer workflows, faster teams

Improved clarity across all three user roles. Agile sprint consistency improved week over week as the team aligned earlier on design intent.

UI / UX Design iOS Development Agile / Scrum

Learnings

What I took away

01

Designing for multiple user roles requires systems thinking, not just visual design.

02

In complex systems, every added element must justify its presence — simplicity is a discipline.

03

Early design feedback in Agile cycles prevents engineering rework — show, don't describe.
